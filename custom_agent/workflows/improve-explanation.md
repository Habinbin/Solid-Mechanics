# Improve Explanation (Typst Material Refinement)

## Prerequisites

1. 사용자가 특정 `.typ` 파일(예: 교재, 강의 자료) 내의 특정 개념, 수식, 문장 등에 대해 질문을 제시해야 합니다.

## Steps

1. **질문 분석 및 답변 도출**:
   사용자의 질문 의도와 불명확했던 맥락을 파악하고, 직관적으로 이해하기 쉬운 답변(비유 추가, 수식 전개 과정 보강, 배경지식 설명 등)을 도출합니다.

2. **교육 자료(Material) 개선 플랜 수립**:
   질문에 대한 답변을 단순히 대화 형식으로만 전달하는 것이 아니라, 원본 `.typ` 파일 자체를 개선할 방법을 설계합니다. 앞으로 다른 학습자가 동일한 의문을 갖지 않도록 어떤 문장을 추가하고, 어떤 순서로 재배치할지 결정합니다.

3. **원고 수정 적용**:
   계획된 개선점을 반영하여 대상 `.typ` 파일의 텍스트를 수정합니다.
   (단, `build-safety.md`를 준수하며 레이아웃이나 기존 수식 포맷팅 등은 보존해야 합니다.)

4. **[Typst] 컴파일 및 빌드 무결성 점검**:
   변경된 `.typ` 파일 문법 오류 없이 정상적으로 빌드되는지 즉시 검증합니다.
// turbo
   `timeout 15 typst compile --root . --font-path 98_Assets/fonts <수정된파일경로>.typ`

5. **자동 Diff 생성**:
   코드의 변경 이력을 명확히 남기기 위해 Typst 파일에 대한 `git diff`를 확인합니다. (참조: `auto-diff-enforcement.md`)
// turbo
   `git --no-pager diff -- <수정된파일경로>.typ`

6. **결과 보고**:
   사용자에게 수정한 내용(어떻게 더 이해하기 쉽게 바꿨는지)을 요약하여 전달하고, 사용자 질문에 대한 최종 답변을 제공합니다.
